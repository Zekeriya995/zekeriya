## 1. Sync git + check ownership

The last session uncovered a `.git/objects` ownership drift
(root-owned files blocked `sudo -u nexus git pull`). Run the
preventive chown EVERY deploy from now on — it's cheap and
removes a known failure mode:

```bash
sudo chown -R nexus:nexus /root/zekeriya/.git
sudo -u nexus git -C /root/zekeriya fetch origin main
sudo -u nexus git -C /root/zekeriya log --oneline HEAD..origin/main
```

The `log` line shows what new commits arrive with the pull. If
it looks right, pull:

```bash
sudo -u nexus git -C /root/zekeriya pull origin main
```

Expect a Fast-forward (no merge commit, no conflicts).

## 6. Rollback (escalating)

If any verification fails, escalate in order:

### 6.1 PWA-only rollback (Phase 2.A.2.x family)

```js
// In browser console — any client device
localStorage.setItem('nxScannerFix_server_signals', 'off');
location.reload();
```

Reverts the bounds overlay + tier demotion to pure-local
scanner behaviour. Server keeps tier-aware ATR multipliers
running (push notifications still get the new TP1) but the
visible card uses client values.

### 6.2 Server tier-aware ATR rollback (PR #110)

```bash
sed -i 's/^SCANNER_TIER_AWARE_ATR_ZONES=true/SCANNER_TIER_AWARE_ATR_ZONES=false/' /root/zekeriya/.env
sudo -u nexus pm2 restart nexus-proxy --update-env
```

Reverts the server's tier-aware multipliers. BTC pushes go
back to wider TP. PWA continues to read the server's bounds
(via overlay) but they're now non-tier-1 values.

### 6.3 Hard rollback — revert merges

```bash
# Get the merge SHAs since the last known-good main
sudo -u nexus git -C /root/zekeriya log --oneline --merges main
# Then for each problem PR:
sudo -u nexus git -C /root/zekeriya revert -m 1 <merge-sha> --no-edit
sudo -u nexus git -C /root/zekeriya push origin main
sudo -u nexus pm2 restart nexus-proxy --update-env
```

Do NOT use `--force` push under any circumstances. Reverts are
new commits.

## 8. Known issues / watch-outs

- **PR #110 user-visible effect:** BTC TP1 drops from +6.5% to
  +3.92% on the same setup. Some users may interpret this as a
  signal weakening. The R:R is still ≥1.3 (the audit's
  profitability floor). Document this in any user-facing release
  note.
- **PR #112 schema dependency:** if the server ever renames `sl`
  / `tp1` / `tp2` / `rr` fields in `/api/all signals[]`, the
  client overlay silently no-ops (numeric gates fail). Both PR
  reviewers flagged this; the gate behaviour is by design but
  worth a console.warn telemetry add in a future PR.
- **PR #114 mutual exclusion gate:** TIER2_BONUS now has a
  precedence check (`isTier1 !== true`) added in the BLOCKER
  fix commit. Without it, a TIER1∩TIER2 overlap would score
  +15 instead of +10. The test suite pins this.
